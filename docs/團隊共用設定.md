# 團隊共用雲端設定指南

預設情況下，App 的「確認清單、對手知識庫、搜尋歷史」都存在**各自的瀏覽器**（localStorage），
換裝置或換人就看不到。啟用「團隊共用雲端」後：

- ✅ 全公司共用**同一份確認清單**與**對手知識庫**
- ✅ 跨裝置、跨帳號同步（換電腦也在）
- ✅ **自我學習**：任何人手動確認/修正過的對照，之後全公司查同一型號都會**優先採用**（推薦第一筆標「團隊已確認」）

> 不設定也完全沒關係——App 會自動用各自瀏覽器的 localStorage 照常運作。

---

## 設定步驟（約 3 分鐘，只需做一次）

這套用的是 **Vercel Storage 的 Upstash Redis**（免費方案額度充足）。

1. 登入 [Vercel](https://vercel.com) → 進入你的 AirTAC 專案
2. 上方分頁點 **Storage** → **Create Database**
3. 選 **Upstash for Redis**（或 KV）→ 取個名字（如 `airtac-team`）→ **Create**
4. 建立後點 **Connect Project**，選你的 AirTAC 專案、環境勾 **Production**（可全勾）
5. Vercel 會**自動注入**連線環境變數（`KV_REST_API_URL`、`KV_REST_API_TOKEN`），你**不用手動填**
6. 到 **Deployments** → 最新一筆 → **Redeploy**（讓新環境變數生效）

完成後打開 App，右上角會出現 **☁ 團隊共用** 徽章，代表已啟用。

---

## 怎麼確認生效？

- App 右上角有 **☁ 團隊共用** 徽章
- 「確認清單」分頁底部顯示「**雲端共用模式**」
- 用瀏覽器開 `https://你的網址/api/store?kind=status`，應回傳 `{"configured":true,"backend":"redis"}`

---

## 自我學習怎麼運作？

1. 業務分析競品型號 → AI 給推薦訂購碼
2. 業務檢查/修正後按「**確認並加入清單**」
3. 系統自動把「競品型號 → 你確認的 AirTAC 碼」記進雲端
4. 之後**任何同事**查同一個競品型號，這筆答案會**直接出現在推薦第一位**（標「團隊已確認」100%）

等於整個團隊的選型經驗會累積下來，用越久越準，錯誤只需修正一次。

---

## 隱私與資料

- 資料存在你自己的 Upstash/Vercel 帳號下，Anthropic/本工具作者無法存取
- 確認清單仍可隨時匯出 Excel 備份
- 若想停用共用，移除 Vercel 的資料庫連結即可，App 會自動退回本機模式
